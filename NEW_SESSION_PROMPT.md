## 프롬프트

```
OKKY 바이브 코딩 해커톤 (2026.02.21) 프로젝트를 개발할 거야.

## 프로젝트: Review Detective (AI 가짜 리뷰 판별기)

쿠팡/네이버 쇼핑 상품 URL을 입력하면, AI가 리뷰를 분석하여:
1. 가짜 리뷰를 판별 (체험단, AI생성, 템플릿형, 평점불일치 등 유형별 분류)
2. 가짜를 제외한 보정 평점 산출
3. 진짜 리뷰만 기반으로 장단점 요약

## 기술 스택
- Next.js 14 (App Router) + TypeScript + Tailwind CSS
- Puppeteer + puppeteer-extra-plugin-stealth (크롤링)
- Claude API (리뷰 분석)
- Recharts (차트)

## 아키텍처
URL 입력 → Puppeteer로 상품 페이지 크롤링 (리뷰 텍스트/평점/날짜/작성자 수집)
→ 리뷰를 10-20개씩 배치로 Claude API에 전달 (가짜 의심도 스코어링)
→ 진짜 리뷰만 필터링 → 요약 생성
→ 대시보드 UI에 결과 렌더링

## 핵심 기능
1. 쿠팡 리뷰 크롤러 (Puppeteer stealth, 페이지네이션으로 100-200개 수집)
2. AI 배치 분석 (리뷰별 fake_score 0-100 + category + reason)
3. 진짜 리뷰 요약 (장점/단점/한줄평/추천대상)
4. 결과 대시보드 (표시평점 vs 보정평점, 도넛차트, 타임라인, 개별 리뷰 리스트)

## 대시보드 출력 항목
- 표시 평점 vs 보정 평점 비교
- 의심 리뷰 비율 (%)
- 카테고리별 분류: genuine / suspected_paid / suspected_ai / suspected_template / rating_mismatch
- 날짜별 리뷰 + 가짜 비율 타임라인 차트
- 진짜 리뷰 기반 장단점 요약
- 개별 리뷰 리스트 (각 리뷰 옆에 의심도 뱃지 + 판단 근거)

## UI 디자인
- 다크 모드 기본
- 숫자는 크고 한눈에 들어오게
- 차트 애니메이션 포함
- 단계별 로딩: "리뷰 수집 중..." → "AI 분석 중..." → "결과 생성 중..."

## 제약 조건
- 해커톤 4시간 내 완성해야 함 (MVP 우선)
- 크롤링 실패 시 fallback 캐시 데이터 지원 필요
- LLM 호출은 배치 + 병렬로 최적화 (속도 중요)
- 데모에서 터지면 안 됨 — 안정성 최우선

상세 구현 플랜은 PLAN.md를 읽어줘. 이 플랜에 따라 순서대로 구현해줘.
```
