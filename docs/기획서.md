# KillMyIdea — 해커톤 아이디어 검증기 개발 기획서

## 1. 문제 정의

"해커톤에서 4시간을 투자했는데, 이미 똑같은 게 있었다."

해커톤 참가자의 가장 큰 리스크는 아이디어 검증 없이 개발에 돌입하는 것입니다. 기존 아이디어 검증 도구들은 비즈니스/스타트업 관점(시장 규모, 수익 모델)에 집중하지만, 해커톤 참가자에게 필요한 것은 "이거 4시간 안에 만들 수 있는가?", "GitHub에 이미 있는가?" 같은 개발자 관점의 검증입니다.

KillMyIdea는 아이디어를 입력하면 웹 검색 + GitHub 코드 탐색 + AI 분석을 자동 수행하여, GO / PIVOT / KILL 판정을 내리는 해커톤 특화 검증기입니다.

## 2. 핵심 기능 (5단계 파이프라인)

| 단계 | 기능 | 데이터 소스 |
|------|------|-------------|
| Step 1 | 경쟁 제품 탐색 | Claude web_search (내장 웹 검색) |
| Step 2 | 유사 오픈소스 심층 분석 | GitHub Search API (3전략 병렬 + README/커밋 분석) |
| Step 3 | 기술 실현성 분석 | Claude API + web_search (API 가용성 실시간 검증, 크롤링 가능 여부, 구현 단계 분해, 복잡도 5축 평가) |
| Step 4 | 차별화 분석 (Devil's Advocate) | Claude API (기능 중복 비교, 시장 틈새 도출) |
| Step 5 | 종합 판정 | Claude API (가중 점수: 실현성 35% + 차별화 25% + 경쟁 20% + 타이밍 20%) |

판정 기준은 가중 평균 70점 이상이면서 모든 카테고리가 25점 이상이면 GO, 40~69점이면 PIVOT, 40점 미만이면 KILL입니다.

## 3. AI 도구 활용 계획

개발 도구로 Claude Code(claude.ai/code)를 사용합니다. 전체 프론트엔드/백엔드 코드 생성, 디버깅, 리팩토링에 활용하며, 바이브 코딩으로 4시간 내 MVP 완성을 목표로 합니다.

서비스 내 AI 활용은 다음과 같습니다.

Claude API(anthropic SDK)가 핵심 엔진입니다. 웹 검색(web_search 내장 도구)과 AI 분석(실현성/차별화/판정)을 총 4회 호출합니다. LLM 출력을 순수 JSON으로 제한하여 파싱 안정성을 확보하고, 응답 실패 시 점수 기반 자동 fallback을 수행합니다. Tavily 등 별도 검색 API 없이 Claude 하나로 통합하였습니다.

GitHub Search API는 3가지 검색 전략(키워드/최근활동/README)을 병렬 실행하고, 상위 레포의 README와 커밋 이력을 자동 수집합니다.

## 5. 시스템 아키텍처

```
[React 19 + Vite]  ──SSE──▶  [FastAPI]  ──▶  Claude API (웹 검색 + AI 분석 ×4)
    (프론트엔드)                (백엔드)   ──▶  GitHub API (코드 탐색)
```

스트리밍 UX는 SSE(Server-Sent Events)로 각 단계를 실시간 표시하며, 로딩 스켈레톤을 사용합니다.

안정성 측면에서는 모든 외부 API 실패 시 graceful fallback을 적용하고, LLM 응답에 대한 JSON 스키마 검증을 수행합니다.

기술 스택은 프론트엔드에 React 19 + TypeScript + Tailwind, 백엔드에 FastAPI + Python을 사용합니다.
