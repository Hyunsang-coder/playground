# KillMyIdea — 해커톤 아이디어 검증기 개발 기획서

## 1. 문제 정의

**"해커톤에서 4시간을 투자했는데, 이미 똑같은 게 있었다."**

해커톤 참가자의 가장 큰 리스크는 아이디어 검증 없이 개발에 돌입하는 것이다. 기존 아이디어 검증 도구들은 비즈니스/스타트업 관점(시장 규모, 수익 모델)에 집중하지만, 해커톤 참가자에게 필요한 건 **"이거 4시간 안에 만들 수 있나?"**, **"GitHub에 이미 있나?"** 같은 개발자 관점의 검증이다.

**KillMyIdea**는 아이디어를 입력하면 웹 검색 + GitHub 코드 탐색 + AI 분석을 자동 수행하여, **GO / PIVOT / KILL** 판정을 내리는 해커톤 특화 검증기다.

## 2. 핵심 기능 (5단계 파이프라인)

| 단계 | 기능 | 데이터 소스 |
|------|------|-------------|
| Step 1 | 경쟁 제품 탐색 | Claude web_search (내장 웹 검색) |
| Step 2 | 유사 오픈소스 심층 분석 | GitHub Search API (3전략 병렬 + README/커밋 분석) |
| Step 3 | 기술 실현성 분석 | Claude API (구현 단계 분해, API 의존성, 복잡도 5축 평가) |
| Step 4 | 차별화 분석 (Devil's Advocate) | Claude API (기능 중복 비교, 시장 틈새 도출) |
| Step 5 | 종합 판정 | Claude API (가중 점수: 실현성 35% + 차별화 25% + 경쟁 20% + 타이밍 20%) |

**판정 기준**: 가중 평균 70+ AND 모든 카테고리 25+ → GO / 40-69 → PIVOT / 40 미만 → KILL

## 3. AI 도구 활용 계획

### 개발 도구
- **Claude Code (claude.ai/code)**: 전체 프론트엔드/백엔드 코드 생성, 디버깅, 리팩토링에 활용. 바이브 코딩으로 4시간 내 MVP 완성 목표.

### 서비스 내 AI 활용
- **Claude API** (anthropic SDK): 핵심 엔진. 웹 검색(web_search 내장 도구) + AI 분석(실현성/차별화/판정) 4회 호출. LLM 출력을 순수 JSON으로 제한하여 파싱 안정성 확보. 응답 실패 시 점수 기반 자동 fallback. **Tavily 등 별도 검색 API 없이 Claude 하나로 통합.**
- **GitHub Search API**: 3가지 검색 전략(키워드/최근활동/README) 병렬 실행 + 상위 레포 README·커밋 이력 자동 수집.

## 5. 시스템 아키텍처

```
[React 19 + Vite]  ──SSE──▶  [FastAPI]  ──▶  Claude API (웹 검색 + AI 분석 ×4)
    (프론트엔드)                (백엔드)   ──▶  GitHub API (코드 탐색)
```

- **스트리밍 UX**: SSE(Server-Sent Events)로 각 단계 실시간 표시, 로딩 스켈레톤
- **안정성**: 모든 외부 API 실패 시 graceful fallback, LLM 응답 JSON 스키마 검증
- **기술 스택**: React 19 + TypeScript + Tailwind (FE) / FastAPI + Python (BE)
